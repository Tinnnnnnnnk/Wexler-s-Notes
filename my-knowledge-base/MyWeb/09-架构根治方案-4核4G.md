# 架构根治方案（4核4G可落地）

## 1. 结论先说

可以完成你的需求，而且不需要上重型后端。

根治方向不是"继续堆特效"，而是把现有站点改造成：

1. 文档内核层（VitePress，负责 md 内容渲染）
2. 首页交互层（Home Lab，负责复杂视觉与交互）
3. UI 系统层（统一状态、层级、动画、性能降级）

这套方案能在 4核4G 服务器稳定运行，因为最终仍然是静态站点 + Nginx，服务器不做重计算。

---

## 2. 当前根因（为什么会"越改越乱"）

你现在的痛点不是单一 Bug，而是"体系冲突"：

1. 全局类名驱动过多：多个组件都在改 `document.documentElement.classList`。
2. 样式文件过大且历史规则叠加：新旧选择器互相覆盖，层级难推断。
3. 动画系统混用：`reveal-*`、模式切换动画、玻璃/液态动画并行，状态边界不清晰。
4. 层级协议缺失：`z-index` 分散定义，容易出现"内容存在但被覆盖"。
5. 首页与文档页耦合：本应只影响首页的样式影响到全站。

---

## 3. 目标架构（不换 VitePress 的前提下）

### 3.1 模块边界

1. `docs/.vitepress/theme/stores/*`
   - 统一状态源，所有写 html class 的逻辑必须通过这里
   - `uiModeState.js`：FX 特效模式、布局模式、性能模式三合一单点写入
   - `editorState.js`：页面编辑器状态（画布块、草稿/发布、审计日志）
   - **命名规范**：统一使用 `*State.js` 后缀，避免 `*Store.js` 与 `*State.js` 混用造成混乱

2. `docs/.vitepress/theme/components/core/*`
   - 全局基础能力：命令面板（CommandPalette）、阅读增强（ReadingEnhancer）、文档体验增强（DocExperienceEnhancer）

3. `docs/.vitepress/theme/components/home-lab/*`
   - 只给首页使用：布局切换、风格切换、背景层（含液态播放器内嵌）
   - `HomeFxBackdrop.vue`：背景层 + 液态播放器（BGM + 波形动画）合并为一个组件，不再拆分
   - `HomeFxToggle.vue`：FX 模式切换按钮组
   - `HomeLayoutToggle.vue`：布局模式切换按钮组
   - `HomeLayoutScenes.vue`：三个布局场景（minimal/dashboard/editorial）容器

4. `docs/.vitepress/theme/components/editor/*`
   - 编辑器能力：画布交互、状态、导入导出
   - `EditableHomeCanvas.vue`：画布块渲染、拖拽/缩放、属性面板
   - `PageEditorToggle.vue`：编辑器开关与鉴权

5. `docs/.vitepress/theme/styles/*`
   - 分层样式，不再只靠一个超长 `style.css`

### 3.2 推荐目录

```text
my-knowledge-base/docs/.vitepress/theme/
├─ index.js
├─ stores/
│  ├─ uiModeState.js          # 三模式单点写入（原 homeFxState + homeLayoutState 合并）
│  └─ editorState.js          # 编辑器状态（已有，保持不变）
├─ components/
│  ├─ core/
│  │  ├─ CommandPalette.vue
│  │  ├─ ReadingEnhancer.vue
│  │  └─ DocExperienceEnhancer.vue
│  ├─ home-lab/
│  │  ├─ HomeFxBackdrop.vue   # 含液态 BGM 播放器
│  │  ├─ HomeFxToggle.vue
│  │  ├─ HomeLayoutToggle.vue
│  │  └─ HomeLayoutScenes.vue
│  └─ editor/
│     ├─ EditableHomeCanvas.vue
│     └─ PageEditorToggle.vue
├─ styles/
│  ├─ tokens.css              # CSS 变量：颜色、圆角、z-index token
│  ├─ layers.css              # z-index 层级映射（--z-bg / --z-content / ...）
│  ├─ base.css                # 重置 + 基础选择器 + 字体
│  ├─ home-lab.css            # 首页专属：布局场景 + FX 特效 + 播放器
│  ├─ effects.css             # 全局动效规范 + reveal 动画 + 主题过渡
│  └─ editor.css              # 编辑器专属：画布 + 工具栏 + 面板
└─ style.css                  # 仅 @import 聚合，不堆业务规则
```

### 3.3 关于 stores 架构的补充说明

现有代码存在状态分散问题：

- `homeFxState.js`：管理 `fxMode`（default/glass/liquid）+ localStorage
- `homeLayoutState.js`：管理 `layoutMode`（minimal/dashboard/editorial）+ localStorage
- `HomeFxBackdrop.vue`、`HomeFxToggle.vue`、`HomeLayoutToggle.vue` 都各自读写 `document.documentElement.classList`

**重构后**，只允许 `uiModeState.js` 一个模块操作 html class。所有组件只读状态，通过 `setFxMode()` / `setLayoutMode()` 等方法修改。

> **注意**：`editorState.js` 体积较大（1765行），本次重构**不移动文件位置**（保持 `stores/editorState.js`），仅确保其不对 html class 写入与 FX/布局状态冲突。

## 4. 统一规范（从根源防屎山）

### 4.1 状态规范

只保留一个 UI 模式入口（单向数据流）：

1. `mode.fx`: `default | glass | liquid`
2. `mode.layout`: `minimal | dashboard | editorial`
3. `mode.perf`: `normal | safe`

禁止多个组件直接改 html 全局类。`uiModeState.js` 是唯一写入口。

### 4.2 层级规范（必须固定）

```text
--z-bg: 0               背景层（HomeFxBackdrop 等）
--z-content: 10          页面内容主层
--z-nav: 40              顶部导航
--z-overlay: 60          功能浮层（章节导航面板等）
--z-editor: 80           编辑器画布层
--z-modal: 100           弹层/遮罩（CommandPalette / Lightbox）
--z-transition: 200      主题切换动画覆盖层
```

任何新组件必须引用 token，不得随意写魔法数字。

### 4.3 动画规范

**时长三档（强制）：**
- `--motion-fast: 120ms` — hover、active 等微交互
- `--motion-mid: 240ms` — 面板展开/收起、模式切换
- `--motion-slow: 420ms` — 页面级进场/退场

**缓动曲线：**
- `--ease-smooth: cubic-bezier(0.22, 1, 0.36, 1)` — 强调手感
- `--ease-soft: cubic-bezier(0.2, 0.8, 0.2, 1)` — 强调弹性

**渲染性能约束：**
1. 大面积动效只允许 `transform` / `opacity`
2. `backdrop-filter` 限制在单个卡片内，禁止在背景层做全屏 blur 动画
3. 检测低性能时自动进入 `safe` 模式：关闭 blob 动画、减少 blur 值
4. 滚动/拖拽 resize 统一使用 `requestAnimationFrame` 节流

### 4.4 样式隔离

1. 首页样式统一挂在 `.home-lab` 命名空间
2. 文档页样式统一挂在 `.VPDoc` 命名空间
3. 禁止裸选择器直接改 `.Layout`、`.VPHome`、`.VPContent`（除 tokens 层）
4. 每个 CSS 文件顶部注释说明自己负责哪部分样式

## 5. 分阶段实施计划

### Phase 0：基线冻结（0.5 天）

1. 给当前可用版本打 tag：`baseline-home-lab`
2. 导出当前 `style.css`、`theme/index.js`、核心组件备份
3. 建立 smoke 检查清单（首页/文档页/移动端/三模式切换）

验收：能一键回退到基线版本。

### Phase 1：目录与状态重构（1 天）

1. 建立 `stores/uiModeState.js`，合并 `homeFxState` 和 `homeLayoutState` 的状态读写逻辑
2. `uiModeState.js` 接管所有 `document.documentElement.classList` 写入
3. 把组件移动到 `components/core/`、`components/home-lab/`、`components/editor/`
4. 更新所有组件的 import 路径
5. `theme/index.js` 只负责装配，不承载业务逻辑

验收：三模式切换可用；不存在双写 class 的逻辑。

### Phase 2：样式分层重构（1~1.5 天）

1. 抽 `styles/tokens.css`（颜色/圆角/z-index/动效变量）
2. 抽 `styles/layers.css`（z-index token 映射）
3. 抽 `styles/base.css`（全局重置 + 字体 + 基础排版）
4. 抽 `styles/home-lab.css`（首页专属：布局场景 + FX 特效）
5. 抽 `styles/effects.css`（reveal 动画 + 主题切换过渡 + 全局动效规范）
6. 抽 `styles/editor.css`（编辑器专属：画布 + 工具栏 + 面板）
7. `style.css` 只做 import 聚合：

```css
@import './styles/tokens.css';
@import './styles/layers.css';
@import './styles/base.css';
@import './styles/home-lab.css';
@import './styles/effects.css';
@import './styles/editor.css';
```

验收：删除旧大段规则后页面行为不变。

### Phase 3：层级冲突治理（0.5~1 天）

1. 全量替换散落 `z-index` 为 token
2. 去掉首页与文档页互相影响规则
3. 增加"层级断言样式"（调试开关）

验收：不存在"可复制但看不见"面板。

### Phase 4：性能治理（1 天）

1. FPS probe 逻辑已在 `HomeFxBackdrop.vue` 中实现，需迁移至 `uiModeState.js` 统一调用
2. 所有模式（default/glass/liquid）切换时均执行性能检测
3. 滚动、拖拽、resize 全部 `requestAnimationFrame` 节流
4. 合并多处 `MutationObserver` 重复监听 html class 为单一 watch

验收：模式切换与滚动不卡顿，CPU 峰值明显下降。

### Phase 5：发布与回滚体系（0.5 天）

1. CI 加两步：`docs:build` + 关键页面快照检查
2. 生产部署保留上一个版本目录（秒级回滚）
3. 记录 `CHANGELOG-UI.md`

验收：发布失败可自动回退。

## 6. 4核4G 资源预算

### 6.1 生产运行（静态站点）

1. Nginx：几十 MB ~ 200MB（常见）
2. 静态资源：磁盘占用为主，内存压力低
3. 并发瓶颈主要来自带宽，不是 CPU/内存

结论：4核4G 充足。

### 6.2 构建策略

1. 构建放 GitHub Actions（推荐）
2. 服务器只接收构建产物（rsync）

结论：服务器负担最小。

### 6.3 若启用在线编辑后端

推荐轻量方案：

1. `Fastify + SQLite` 单容器
2. 不上 Redis/ES
3. 加一个简单鉴权（JWT 或 Basic Auth）

4核4G 仍可承载个人站点规模。

---

## 7. 风险与规避

1. 风险：重构期间样式回归
   - 规避：每 phase 都做页面快照 + smoke 测试

2. 风险：模式状态串线
   - 规避：只有 `uiModeState.js` 可写模式

3. 风险：历史 CSS 残留
   - 规避：建立"废弃规则清单"，逐项删除并验证

4. 风险：上线后缓存导致"看起来没改"
   - 规避：静态资源 hash + 强缓存策略 + 版本号提示

---

## 8. 验收标准（完成即算根治成功）

1. 三种布局切换后不闪烁、不消失
2. 三种特效切换后无叠层遮挡
3. 首页样式改动不影响文档页
4. 文档页增强不影响首页布局
5. 代码结构可定位：任何一个问题 5 分钟内能找到责任模块

---

## 9. 执行说明

本方案可直接执行，Phase 1 + Phase 2 最关键，Phase 3 解决你最近遇到的遮挡/闪现问题，Phase 4 + Phase 5 完善性能和交付闭环。
