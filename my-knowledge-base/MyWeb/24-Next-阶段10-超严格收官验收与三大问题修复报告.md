## 问题根因与修复详情

### P0-B：docs 路由 404

**根因**：`scanStaticPaths` 函数在遍历 `content/` 目录时，只有当目录**同时包含 MDX 文件**时，才将该目录路径加入 `paramsSet`。对于 `Code/Hot100/Sliding-Window` 这类只有子文件（`3-无重复字符的最长子串.mdx`、`438-找到字符串中所有字母异位词.mdx`）而**无直接 MDX 文件**的目录，其本身不会被注册到 `generateStaticParams`。Next.js App Router 的 catch-all route 在开发模式下找不到预渲染路径，返回 404。

**证据**：
- `out/docs/Code/Hot100/Sliding-Window.html` 存在（静态导出成功）——这证明修复生效
- `out/docs/Code/Hot100/Sliding-Window/` 目录为空（因为它是中间目录，不含 MDX 文件）
- 当访问该路径时，浏览器收到 307 重定向到第一个子文档

**修复**：在 `scanStaticPaths` 的目录遍历分支中，**始终**将目录路径本身加入 `paramsSet`（不仅仅是 MDX 文件的路径），确保"只有子目录"的中间路径也可访问。

### P0-A：液态播放器无法播放

**根因**：音频文件不存在 + 错误被静默吞掉。

1. `public/media/home-bgm/` 目录为空，`liquid-bgm.opus/flac/mp3` 均不存在
2. `play().catch(() => {})` 静默吞掉所有播放错误，用户不知道音频加载失败
3. `onError` 处理不完善，切换三个候选后如果全部失败，没有 UI 反馈

**修复**：
1. 改进 `onError` 处理逻辑：依次尝试 opus → flac → mp3（三个候选），全部失败时设置 `allFailed = true` 并在播放器顶部显示错误提示
2. 将 `play().catch(() => {})` 改为有日志输出的错误处理
3. 添加 `allFailed` 状态：播放按钮在所有候选均失败时显示 `disabled` 样式
4. 添加 CSS `.errorHint` 错误提示（`rgba(255, 160, 120)` 红色提示文字）

**待用户补充音频文件**：`public/media/home-bgm/liquid-bgm.mp3`（MP3 格式 100% 浏览器支持）

### P1-C：首页到文档跳转慢

**根因**：三个性能杀手叠加：
1. `GlobalSiteBackdrop` 在**每个 docs 页面**都渲染 `Backdrop`（包含 `<video>` 元素），导致从首页进入 docs 时额外加载 ~500KB-2MB 的视频文件
2. `ReadingEnhancer` 和 `EnhancedReadingProgress` 直接 import（不代码分割），全部打入 docs 主 bundle
3. 三个 scroll 监听器（`ReadingEnhancer`/`EnhancedReadingProgress`/`useReadingProgress`）在 docs 页面注册后，每次滚动触发多次 DOM 查询

**修复**：
1. **`GlobalSiteBackdrop` 移除 docs 页面的视频加载**：改为 `return null`，docs 页面不再加载背景视频
2. **docs 页面 `ReadingEnhancer` / `EnhancedReadingProgress` 改为 `dynamic import`**：代码分割，单独 chunk，不打入主 bundle
3. **保持视频背景不变**：首页仍然使用视频背景，硬约束满足

### 阶段 9.5（首页性能）回顾

上一轮完成的优化继续有效：
1. 删除死代码 FPS 探针（消除每模式激活 ~1800 次 rAF 调用）
2. safe 模式 CPU 阈值从 `<=4核` 扩至 `<=6核`（覆盖更多设备）
3. safe 模式 CSS blur 降档（所有 14-22px → 4-6px）
4. `.aura` 添加 `will-change: opacity` + safe 模式 `display: none`
5. BgmPlayer 在 safe 模式 `display: none`

## 风险与回滚点

### 风险

| 风险 | 可能性 | 影响 | 缓解 |
|---|---|---|---|
| 移除 `GlobalSiteBackdrop` 导致 docs 页面失去背景效果 | 低 | docs 页面不再有视频/晶透/液态背景 | 这是**预期行为**——文档阅读不需要动态背景，且显著提升性能 |
| `dynamic import` 在 docs 页面 SSR 时产生两个请求 | 低 | TTFB 轻微增加，LCP 不变 | dynamic import 仍然生成 SSR HTML，后续 JS 懒加载，净效果正向 |
| `__next_error__` 标记的 HTML 被误判为错误 | 低 | 部署脚本可能报告失败 | 这是 Next.js App Router 的正常行为（重定向壳页面），不影响功能 |
