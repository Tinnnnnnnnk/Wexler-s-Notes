# 🌲 广度优先搜索 (BFS) 万能解题模板 (网格层序扩散)

> [!info]- 题目描述 / 适用场景 (点击展开)
> > **核心适用场景：**
> > 1.  **求最短路径/最少步数**（例如：迷宫的最短出路、单机游戏的最少操作次数）。
> > 2.  **层序扩散/多源并发**（例如：腐烂的橘子、水波蔓延、多处着火点同时蔓延）。
> > 
> > **特征识别：** 只要题目暗示了“一圈一圈往外扩”、“按分钟/步数推进”，必须无条件掏出带有 `Queue` 的 BFS！

---


## 💡 解题思路 (Kasumi's Memo)

### 核心策略 (回合制丧尸桌游)
1.  **第一步（初始海选 / 入队）：** 遍历一次全图。把所有“初始感染源（起点）”打包放进隔离区（队列 `Queue`），顺便统计好“平民（需要被感染的目标总数）”，避免最后再去扫图。
2.  **第二步（拉警戒线 / 锁定当前回合）：** 进入 `while(!queue.isEmpty())`。核心动作：立刻获取当前队列长度 `int size = queue.size();`，这根警戒线决定了当前这一分钟只有多少个老丧尸能出来咬人。
3.  **第三步（出击与扩散）：** 开启 `for(int i = 0; i < size; i++)` 循环。放老丧尸出笼，配合十字罗盘（`dirs` 方向数组）向四周探索。
4.  **第四步（新兵入列 / 回合结算）：** 发现目标后，立刻将其标记为“已感染”，并**扔进队列尾部排队**。等 `for` 循环结束，意味着这一分钟结束，时间（步数）加 1。

### 为什么选择这个解法？
- **时间物理隔离：** 严格利用 `size` 快照，将“上一秒的丧尸”和“这一秒刚变异的丧尸”完美隔离开，彻底杜绝“一秒钟感染全图”的时空错乱 Bug。
- **天然的最短路：** BFS 就像水波纹，最先触碰到目标的水波，走过的步数绝对就是最短的。

## 💻 代码实现 (Java)

```java
import java.util.LinkedList;
import java.util.Queue;

class Solution {
    public int bfsTemplate(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        
        // 核心神器：排队隔离区
        Queue<int[]> queue = new LinkedList<>();
        int targetCount = 0; // 目标总数（比如好橘子数）

        // 1. 海选阶段：寻找所有起点入队
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 2) { // 假设 2 是起点/初始丧尸
                    queue.offer(new int[]{i, j});
                } else if (grid[i][j] == 1) { // 假设 1 是需要被转化的目标
                    targetCount++;
                }
            }
        }

        // 防御性判断
        if (targetCount == 0) return 0;

        int step = 0; // 回合数/分钟数
        // 魔法罗盘：上下左右走位
        int[][] dirs = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

        // 2. 回合制扩散开始
        while (!queue.isEmpty()) {
            // 🌟 灵魂一步：拉起警戒线，锁死本回合能出动的老丧尸数量！
            int size = queue.size();
            boolean hasSpread = false; // 本回合是否真的扩散了

            // 3. 处理本回合的老丧尸
            for (int i = 0; i < size; i++) {
                int[] curr = queue.poll();
                int row = curr[0], col = curr[1];

                // 4. 朝四周咬人
                for (int[] dir : dirs) {
                    int r = row + dir[0];
                    int c = col + dir[1];

                    // 如果没越界，且找到了合格的目标
                    if (r >= 0 && r < m && c >= 0 && c < n && grid[r][c] == 1) {
                        grid[r][c] = 2; // 标记感染状态（极其重要，防止互相反复感染）
                        queue.offer(new int[]{r, c}); // 新丧尸入队排到最后
                        targetCount--; // 目标减少
                        hasSpread = true;
                    }
                }
            }
            
            // 本回合所有老丧尸处理完毕，如果咬到了人，时间才往前走
            if (hasSpread) {
                step++;
            }
        }

        // 5. 终局结算：检查目标是否全部被感染完
        return targetCount == 0 ? step : -1;
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(M \times N)$
  - *分析：* 最坏情况下，网格中的每个格子最多只会入队一次、出队一次，被罗盘检查四周一次。计算量与网格总数严格成正比。
- **空间复杂度：** $O(M \times N)$
  - *分析：* 空间消耗主要在于队列 `Queue`。最坏情况下（例如整张图全都是初始丧尸），队列一开始就会装满所有网格点。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **双层 `for` 循环的“一秒烂全图”怪圈：** 在网格题中直接用两层循环代替 BFS，导致当前遍历刚修改的状态，立马又被紧接着的遍历触发，形成时空错乱。
> > 2. **忘记入队即标记：** 把格子加进 `Queue` 后，如果没有立刻把它的状态修改为“已感染”（如 `grid[r][c] = 2`），其他丧尸的罗盘也会扫到它，导致同一个格子被重复加入队列一百次，直接爆内存（Memory Limit Exceeded）。
> > 3. **手写臃肿的 `if-else`：** 手动写判断上、下、左、右的代码，极易因为笔误写错加减号导致数组越界。

> [!tip] 关键技巧 / 总结
> > **三大法宝：** `Queue` 作为隔离区，`size = queue.size()` 拉起警戒线分离层级，`dirs` 数组作为方向罗盘！只要记住这三板斧，所有网格搜索题瞬间变成机械劳动。

- **🔧 工程思维：** BFS 就是一种典型的“消息队列缓冲区 (Message Queue Buffer)”思想。现实系统中的高并发请求，也是先进缓冲队列，服务器按批次（层）拉取处理，绝不抢跑。
