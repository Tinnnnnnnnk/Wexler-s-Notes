# 🏆 动态规划 (Dynamic Programming) 核心武功秘籍

> [!abstract] 什么是 DP？
> DP 的本质是**“带记事本的递归”**或者**“历史经验的复用”**。
> 它的终极世界观是：**未来某一步的最优解，绝对可以由过去某几步的最优解推导出来。** 只要我们把过去的历史记录在案（`dp` 数组），就能像滚雪球一样推出最终答案。

---

## 🧠 DP 破局的“标准工业 5 步法”

无论是一维还是二维，做 DP 题必须在脑子里走完这 5 步，少一步都会在写代码时走火入魔：

1.  **定义 `dp` 数组的含义 (最重要的一步！)：** 明确 `dp[i]` 或 `dp[i][j]` 到底代表什么（是最大价值？最小硬币数？还是方法总数？）。
2.  **推导“状态转移方程”：** 寻找现在的自己和过去的自己之间的关系。例如：$dp[i] = \max(dp[i-1], dp[i-2] + nums[i])$。
3.  **初始化 `dp` 数组：** 万丈高楼平地起，第 0 步或第 1 步的值必须准确无误（如 `dp[0] = 0`）。
4.  **确定遍历顺序：** 是一维的从左到右？还是二维的从上到下、从左到右？
5.  **模拟打印 (Debug)：** 如果代码出错了，把 `dp` 数组打印出来，和你在纸上推导的对比，一眼就能看出哪一步算错了。

## 🗡️ 第一层境界：一维 DP (1D DP)

一维 DP 通常用来解决**“单序列”**问题。历史状态只跟“前面几个位置”或者“容量”有关。

### 1. 状态机模型 (代表作：打家劫舍)
- **特点：** 每个位置上有不同的选择（偷 or 不偷）。
- **你的代码复盘：** `dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]);`
  - *翻译：* 走到第 $i$ 个房间，我面临两个选择。要么不偷（保持前一个房间的最高收益 `dp[i-1]`）；要么偷（拿走现在的钱，加上大前个房间的收益 `dp[i-2] + nums[i]`）。取两者最大值！

### 2. 背包模型变种 (代表作：零钱兑换、完全平方数)
- **特点：** 给定一个容量（目标金额），用一堆物品（硬币、平方数）去填满它。
- **你的代码复盘：** `dp[i] = Math.min(dp[i], dp[i - coins[j]] + 1);`
  - *翻译：* 我想凑出金额 $i$。如果我决定用面值为 `coins[j]` 的这枚硬币，那我只需要知道“凑出金额 $i - coins[j]$ 的最少硬币数”，然后再加 1（也就是手里这枚硬币）就行了。

## ⚔️ 第二层境界：二维 DP (2D DP)

当我们面对的是**“网格地图”**、**“两个字符串的匹配”**，或者**“极其复杂的背包限制”**时，一根直线的数组不够用了，我们需要一张二维的表格 `dp[i][j]`。

### 场景一：网格路线问题 (Grid Paths)
- **典型问题：** LeetCode 62. 不同路径 (机器人从左上角走到右下角，有多少种走法)。
- **定义：** `dp[i][j]` 表示走到坐标 $(i, j)$ 的路径总数。
- **转移方程：** 机器人只能向下或向右走。所以，走到 $(i, j)$ 的方法数，等于走到它**上面那格**的方法数，加上走到它**左边那格**的方法数。
  - $dp[i][j] = dp[i-1][j] + dp[i][j-1]$

### 场景二：双字符串匹配 (String Matching)
- **典型问题：** LeetCode 1143. 最长公共子序列 (LCS)。
- **定义：** `dp[i][j]` 表示字符串 $A$ 的前 $i$ 个字符，和字符串 $B$ 的前 $j$ 个字符，它们的最长公共子序列长度。
- **转移方程：** - 如果最后一个字符相等 ($A[i-1] == B[j-1]$)：$dp[i][j] = dp[i-1][j-1] + 1$
  - 如果不相等：$dp[i][j] = \max(dp[i-1][j], dp[i][j-1])$ (要么抛弃 A 的末尾，要么抛弃 B 的末尾，看谁保留的子序列长)。

### 场景三：正宗的 0-1 背包问题 (Knapsack)
- **定义：** `dp[i][j]` 表示从前 $i$ 个物品中任选，放进容量为 $j$ 的背包里，能装出的最大价值。
- **转移方程：** 不选第 $i$ 个物品 vs 选第 $i$ 个物品。
  - $dp[i][j] = \max(dp[i-1][j], dp[i-1][j - weight[i]] + value[i])$

## 🚫 架构师避坑指南

> [!failure] DP 最容易死在哪里？
> 1.  **无穷大溢出陷阱：** 找最小值时，很多人喜欢把 `dp` 数组初始化为 `Integer.MAX_VALUE`。结果在转移方程里做 `+1` 时，直接越界变成负数！
>     *✅ 你的解法极其完美：用 `amount + 1` 或 `100000` 作为无穷大，安全又实用。*
> 2.  **数组越界：** 在写 `dp[i-1]` 或 `dp[i-2]` 时，永远要先处理好 `i=0` 和 `i=1` 的初始化，并且循环从合适的地方开始。
> 3.  **搞反遍历顺序：** 大部分题是从左往右，但有些背包问题（如一维数组优化的 0-1 背包）必须**从右往左**遍历，防止物品被重复使用。
