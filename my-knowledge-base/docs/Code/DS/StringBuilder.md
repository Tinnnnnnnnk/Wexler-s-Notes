# 🛠️ Java 核心神器：StringBuilder 完全指南

> [!abstract] 核心定义
> `StringBuilder` 是 Java 中用于处理**可变字符串**的核心类。
> 与绝对不可变的 `String` 不同，`StringBuilder` 允许你在**原内存地址**上直接对字符进行追加、插入、删除和修改，极大地节省了内存开销并提升了运行速度。



---

## 🧠 底层原理解析 (为什么它这么快？)

理解 `StringBuilder`，只需要记住它的两个核心物理组件：
1. **`char[] value` (字符数组)：** 它的肚子里藏着一个真实的、长度比实际内容要长一些的字符数组（通常叫作容量 Capacity）。
2. **`int count` (指针/计数器)：** 记录当前数组里真正写入了几个字符。

**光速操作的秘密：**
- **追加 (`append`)：** 只是在 `value[count]` 的位置写下新字符，然后 `count++`。时间复杂度 **O(1)**。只有当数组真的装满了，它才会自动申请一个更大的新数组并把数据搬过去（扩容）。
- **删尾巴 (`deleteCharAt`)：** 根本没有物理擦除！只是把指针往回拨一格 `count--`。下次再追加时，新字符会直接覆盖在这个“废弃”的位置上。时间复杂度 **O(1)**。

## ⚔️ 字符串三剑客对比 (高频面试题)

| 特性 | `String` | `StringBuilder` | `StringBuffer` |
| :--- | :--- | :--- | :--- |
| **可变性** | **不可变** (每次修改都会 new 新对象) | **可变** (在原数组上修改) | **可变** (在原数组上修改) |
| **线程安全** | 线程安全 (因为不可变) | **非线程安全** (没有加锁，单线程下速度最快) | **线程安全** (内部方法加了 `synchronized` 锁) |
| **执行效率** | 极低 (频繁拼接时) | **极高** (绝对的性能王者) | 较低 (因为有锁的开销) |
| **适用场景** | 少量字符串操作、作为 HashMap 的 Key | **绝大多数的字符串拼接、算法题** | 多线程并发修改同一个字符串 |

## 💻 核心 API 兵器谱 (常用方法)

在日常开发和算法题中，掌握以下几个方法就足够你横着走了：

### 1. 增 (Append / Insert)
- `sb.append(xxx)`：在尾部追加。支持几乎所有数据类型 (int, char, String, 甚至 Object)。
- `sb.insert(int offset, xxx)`：在指定索引 `offset` 的位置强行插入一段内容，后面的字符会自动后挪。

### 2. 删 (Delete)
- `sb.deleteCharAt(int index)`：删除指定座位上的单个字符。（回溯法最常用的“光速切尾巴”操作就是 `sb.deleteCharAt(sb.length() - 1)`）。
- `sb.delete(int start, int end)`：删除一个区间的内容。**注意：左闭右开 `[start, end)`**。

### 3. 改 (Set / Replace)
- `sb.setCharAt(int index, char ch)`：把指定座位上的字符直接替换成新的字符。
- `sb.replace(int start, int end, String str)`：把指定区间的内容替换成另一段字符串。

### 4. 查与转换 (Get / Convert)
- `sb.charAt(int index)`：查看指定座位上的字符是什么。
- `sb.length()`：获取当前真正装了多少个字符。
- `sb.toString()`：**终极结算！** 将当前的 `StringBuilder` 固化，生成一个真正的 `String` 对象并返回。

### 5. 逆天神技 (Reverse)
- `sb.reverse()`：将整个字符串**原地倒序反转**。这是 `String` 根本没有的方法，遇到反转字符串的题目直接调用，降维打击。

## 📝 最佳实践与避坑指南

> [!tip] 预分配容量 (极致优化)
> 如果你在使用前，**大概知道这个字符串最终会有多长**（比如生成长度为 n 的括号，最终长度是 2n），最好在创建时直接告诉它：
> `StringBuilder sb = new StringBuilder(2 * n);`
> 这样可以避免它在运行过程中因为数组不够大而触发“自动扩容（申请新数组 + 复制老数据）”，把性能压榨到极限。

> [!warning] 误区：不要在 StringBuilder 里面用 `+`
> ❌ 错误示范：`sb.append(a + ":" + b);`
> 这样写会导致内部先生成隐式的 `StringBuilder` 去拼接 `a` 和 `b`，然后再交给外层的 `sb`。
> ✅ 正确示范：`sb.append(a).append(":").append(b);` (链式调用，丝滑且没有任何额外开销)。
