# 🌳 二叉树终极套路：自底向上的状态汇报 (树形 DP)

> [!abstract] 核心思想：老板与分身的终极艺术
> 在处理绝大多数“困难 (Hard)”级别的二叉树问题时，自顶向下（前序遍历）往往行不通，因为父节点在没有看到子节点的全貌时，根本无法做出决策。
> 
> **破局之道：后序遍历。**
> 核心逻辑是：**“我不知道答案，但我派左右分身去算。等他们把子树的报告交上来，我再根据他们的报告，算出我这个节点的终极答案，然后继续往上级汇报。”**
>
> 

---

## 🎯 适用场景 (这类题的共同特征)

当你看到题目要求：
1. **寻找二叉树中的某种“最大/最小”或“最值”路径**（例如：最大路径和、树的直径）。
2. **父节点的答案，严重依赖于左右子树的计算结果**（例如：最近公共祖先、判断平衡二叉树、打家劫舍 III）。
3. **需要在整棵树中寻找某个特定条件的节点，并将其传递到根节点。**

只要符合以上特征，**立刻无脑默写这套后序遍历模板！**

---

## 🛠️ 万能代码模板 (Java)

所有这类题目的代码骨架都是一模一样的，只有**“步骤 3：老板如何做决策”**这里的逻辑会根据具体题目发生变化。

```java
// 返回值类型可以是 int (求深度/路径和), boolean (判断平衡), 或者 TreeNode (找祖先)
public ReturnType dfs(TreeNode root) {
    // 1. 底线拦截 (Base Case)：老板遇到空气，直接返回空气属性
    if (root == null) {
        return 默认值; // 比如 0, true, 或者 null
    }

    // 2. 信任飞跃 (Delegate)：无条件派分身去拿左右子树的情报
    ReturnType leftInfo = dfs(root.left);
    ReturnType rightInfo = dfs(root.right);

    // 🌟 3. 核心决策 (Synthesize - 后序遍历位置)：
    // 此时，老板手里已经握着 leftInfo 和 rightInfo 两份完美报告了。
    // 老板需要思考：用这两份报告，加上我自己 (root.val)，能拼凑出什么有用的信息？
    
    // 【业务逻辑区：根据不同题目写不同的计算逻辑】
    ReturnType myInfo = calculate(leftInfo, rightInfo, root.val);
    // (可选) 如果题目需要记录全局最大值，通常也会在这里进行：
    // globalMax = Math.max(globalMax, ...);

    // 4. 向上级汇报 (Report)：把老板自己算出的结果，继续往上抛
    return myInfo;
}
```

## 🌐 经典应用案例 (降维打击)

这套模板可以秒杀以下所有经典名题：

### 1. 二叉树的直径 (LeetCode 543)
- **左报告：** 左子树的最大深度。
- **右报告：** 右子树的最大深度。
- **老板决策：** 经过我这里的最长直径就是 `left + right`（更新全局最大直径）。
- **向上汇报：** 我这棵树的最大深度是 `Math.max(left, right) + 1`。

### 2. 二叉树中的最大路径和 (LeetCode 124 - Hard)
- **左报告：** 左子树能提供的最大单边贡献值（如果是负数就当作 0）。
- **右报告：** 右子树能提供的最大单边贡献值（同上）。
- **老板决策：** 穿过我这里的最大总和是 `root.val + left + right`（更新全局最大和）。
- **向上汇报：** 我能提供给上级的最大单边贡献是 `root.val + Math.max(left, right)`。

### 3. 判断平衡二叉树 (LeetCode 110)
- **左报告：** 左子树的深度（如果左边已经不平衡了，直接返回 -1）。
- **右报告：** 右子树的深度（如果右边已经不平衡了，直接返回 -1）。
- **老板决策：** 如果 `left == -1` 或 `right == -1` 或 `|left - right| > 1`，说明以我为根的树也不平衡了！
- **向上汇报：** 如果不平衡返回 -1；如果平衡，返回我的深度 `Math.max(left, right) + 1`。

### 4. 最近公共祖先 (LeetCode 236) *(我们刚做的那道)*
- **左报告：** 左边有没有发现 p 或 q。
- **右报告：** 右边有没有发现 p 或 q。
- **老板决策：** 左右都有？那我就是公共祖先！
- **向上汇报：** 把我自己，或者有目标的那一半报告传递上去。

> [!tip] Kasumi's Memo (终极心法)
> 不要试图去大脑里“走迷宫”一样推演递归的每一步！
> 你只需要把自己当成一个**极其冷酷、只看报告结果的老板**。只思考当前节点这一层：“如果左右护法已经把完美的结果交给我了，我该怎么利用这两个结果交差？”
> 只要当前这一层的逻辑对了，整棵树的逻辑就绝对是对的。这就叫**“信任的飞跃 (Leap of Faith)”**。
