# 236. 二叉树的最近公共祖先 (Lowest Common Ancestor of a Binary Tree)

> [!info]- 题目描述 (点击展开)
> > 给定一个二叉树, 找到该树中两个指定节点 `p` 和 `q` 的最近公共祖先。
> > **百度百科中最近公共祖先的定义为：**“对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

---

## 💡 解题思路 (Kasumi's Memo)



### 核心策略 (底向上汇报 / 后序遍历)
彻底抛弃“记录路径再比对”的物理思维，转为**“状态向上传递”**的递归思维。

1.  **第一步（定底线 / Base Case）：** 老板亲自出马。如果当前节点 `root` 是空气（`null`），直接向上级返回 `null`。如果当前节点刚好就是我们要找的 `p` 或者 `q`，直接向上级汇报：“我找到目标了，就是我本人！”（返回 `root`）。
2.  **第二步（派任务 / 信任飞跃）：** 老板派分身去左子树找，拿到左边的汇报结果 `left`；派分身去右子树找，拿到右边的汇报结果 `right`。
3.  **第三步（做决策 / 后序位置）：** 根据左右分身的汇报，做出终极审判：
    - **情况 A (左右开弓)：** 如果 `left` 和 `right` 都有人（都不为空），说明 `p` 和 `q` 分居在我的两侧，**那我本人绝对就是最近公共祖先！** 向上级返回 `root`。
    - **情况 B & C (单传或扑空)：** 如果左右只有一边有人，说明目标全在那一边（或者找到了其中一个），那我就当个传话筒，把有人的那一边直接递给上级 (`return left != null ? left : right`)。

### 为什么选择这个解法？
- **降维打击：** 这是 LCA 问题的最优解。不需要维护任何 `List` 路径数组，不需要繁琐的 `for` 循环比对。空间复杂度从 $O(N)$ 的显式数组降维到了 $O(1)$（仅剩系统栈开销），代码极度优雅。

## 💻 代码实现 (Java)

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        // 1. 递归的尽头，或者直接命中了目标
        if (root == null || root == p || root == q) {
            return root;
        }

        // 2. 无条件信任分身，去左右子树搜寻
        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);

        // 3. 核心决策 (后序遍历：拿着子树的结果做决定)
        // 既然左边找到一个，右边找到一个，那我必然是你们的公共祖先！
        if (left != null && right != null) {
            return root;
        }

        // 否则，哪边找到了目标，我就把哪边的结果继续往上传递
        return left != null ? left : right;
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(N)$
  - *分析：* 最坏情况下（树退化成链表，且目标在最底部），我们需要访问树中的所有节点。
- **空间复杂度：** $O(H)$
  - *分析：* $H$ 为树的高度。不需要额外的 `List` 存储路径，空间消耗纯粹来自于递归调用的系统栈。最坏情况 $O(N)$，平衡树 $O(\log N)$。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **过度劳累的死脑筋：** 试图用回溯法把 `p` 和 `q` 的完整根节点路径全部找出来存进 `List`。这不仅导致了极其频繁的 `ArrayList` 扩容和深拷贝开销，而且在已经找到目标后，程序依然会把剩下的无用节点全部遍历完。
> > 2. **忽略了“包含”关系：** 如果 `p` 刚好是 `q` 的父节点，这种自底向上的代码会在碰到 `p` 的瞬间直接 `return p`，根本不会再去搜 `q`。而这恰好是完全正确的！（因为 `p` 就是最近公共祖先）。

> [!tip] 关键技巧 / 总结
> > **返回值的一语双关：** 这里的返回值既扮演了 `boolean` 的角色（返回 null 代表没找到，返回非 null 代表找到了），又携带了“载荷”（返回的具体节点到底是 p、q，还是确立了身份的祖先）。这种设计是神来之笔。

## 🧠 深度思考与感悟 (Reflections)
- **💡 触类旁通：** 凡是需要在二叉树中“收集左右子树的情报，然后在当前节点做汇总”的题目，绝对都是 **后序遍历** 的主场（例如：求二叉树的最大深度、判断平衡二叉树、求最大路径和）。
- **🔧 工程思维：** “少即是多”。在设计递归函数时，尽量减少对全局变量（如各种全局 List、总和变量）的依赖，让函数变成一个“纯函数”（输入参数，吐出结果）。这样不仅不易写出 Bug，而且天然支持多线程和并发。
- **✨ 瞬间感悟：** 最美的代码，往往不是堆砌数据结构，而是把数学逻辑与控制流（递归的归）完美融合。那个 `return left != null ? left : right` 就像一个自动升降机，把最底层的真相一路保送到了最顶层。
