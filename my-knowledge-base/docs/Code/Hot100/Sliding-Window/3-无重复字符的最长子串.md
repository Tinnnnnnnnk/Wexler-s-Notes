# 3. 无重复字符的最长子串

> [!info]- 题目描述 (点击展开)
> 给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。
> 
> **示例 1:**
> 输入: s = "abcabcbb"
> 输出: 3 
> 解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。注意 "bca" 和 "cab" 也是正确答案。
> 
> **示例 2:**
> 输入: s = "bbbbb"
> 输出: 1
> 解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。
> 
> **示例 3:**
> 输入: s = "pwwkew"
> 输出: 3
> 解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。请注意，你的答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串。
> 
> **提示：**
> * $0 <= s.length <= 5 * 10^4$
> * `s` 由英文字母、数字、符号和空格组成

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略
1.  **第一步：定义滑动窗口与数据结构**
    使用双指针维护窗口 `[pre, last]`。放弃传统的 `HashMap`，改用 **长度为 128 的整型数组** `int[] index` 作为哈希表的替代品（因为字符集是 ASCII）。数组下标是字符 ASCII 码，数组的值是字符 **上一次出现的索引**。
2.  **第二步：窗口滑动与跳跃**
    右指针 `last` 遍历字符串。当检测到当前字符 `now` 已经存在（`index[now] != -1`）时，说明窗口内出现重复。此时左指针 `pre` **不需要一步步移动**，而是直接 **瞬移** 到重复字符的下一位（即 `index[now] + 1`）。

### 为什么选择这个解法？
* **数组优化版** 比 `HashMap` 版本更快。避免了 `Integer` 对象的自动装箱/拆箱开销，也避免了哈希碰撞的计算。这是面试中的加分项（体现对底层和性能的追求）。

## 💻 代码实现 (Java)

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        // 1. 使用数组代替 Map，提升性能 (ASCII 码范围 0-127)
        // index[c] 存储的是字符 c 上一次出现的下标
        int[] index = new int[128];

        // 2. 初始化数组为 -1，表示所有字符都没出现过
        // (不能默认为0，因为第0个位置的字符也可能重复，会导致逻辑混淆)
        for (int i = 0; i < 128; i++) {
            index[i] = -1;
        }

        int max = 0; // 记录最大长度
        int pre = 0; // 窗口左边界 (闭区间)

        // 3. 开始滑动，last 是窗口右边界
        for (int last = 0; last < s.length(); last++) {
            char now = s.charAt(last);

            // 4. 如果当前字符以前出现过，更新左边界
            if (index[now] != -1) {
                // 🌟 核心：pre 直接跳到重复字符的下一位
                // ⚠️ 必须用 Math.max：防止 pre 倒退。
                // 例如 "abba"，遇到第二个 b 时 pre 跳到 2；再遇到第二个 a 时 index['a'] 是 0，
                // 如果不取 max，pre 会退回到 1，导致错误的窗口范围。
                pre = Math.max(pre, index[now] + 1);
            }

            // 5. 记录当前字符的最新位置
            index[now] = last;

            // 6. 每次移动都计算当前窗口长度，并更新最大值
            max = Math.max(max, last - pre + 1);
        }

        return max;
    }
}
```

---

## 📊 复杂度分析
- **时间复杂度：** $O(N)$
  - *分析：* 其中 $N$ 是字符串的长度。虽然有指针跳跃，但右指针 `last` 严格遍历一次字符串，没有嵌套循环，所以是线性的。
- **空间复杂度：** $O(1)$
  - *分析：* 我们申请了一个固定长度为 128 的数组 `index`。无论字符串有多长，这个辅助空间的大小是固定的常数（如果是扩展 ASCII 则是 256），所以是 $O(1)$。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **左指针倒退问题**：在更新 `pre` 时直接用了 `pre = index[now] + 1`，没有加 `Math.max`。这会导致当重复字符出现在当前窗口 **左侧外面**（已经过期的历史记录）时，`pre` 指针错误地往回跳。
> > 2. **数组默认值问题**：`int[]` 默认值是 0。如果字符串第一个字符是重复项，而我又用 0 代表“未出现”，逻辑就会乱。**一定要初始化为 -1**。

> [!tip] 关键技巧 / 总结
> > 1. **Map 转 数组**：在处理字符相关问题时，如果字符集确定（如 ASCII），优先考虑用 `int[128]` 数组代替 `HashMap`，性能提升显著。
> > 2. **跳跃优于蠕动**：能知道确切位置直接跳过去，就不要写 `while` 循环一步步移，这能优化常数级的时间开销。

## 🧠 深度思考与感悟 (Reflections)
- **💡 触类旁通：** 这种“记录上一次出现位置”的思路，在很多**子数组/子串**问题中都通用，比如“和为 K 的子数组”（用前缀和+Map记录位置）。
- **🔧 工程思维：** 虽然 `HashMap` 代码写起来舒服，但在高频调用的底层代码（如解析器、网关）中，利用**基本数据类型 (Primitive Type)** 数组代替对象容器，是极致性能优化的必修课。
- **✨ 瞬间感悟：** 滑动窗口就像是相机的对焦框，与其一点点挪动边框，不如直接根据“前车之鉴”（历史索引）瞬间拉动对焦！
