# 438. 找到字符串中所有字母异位词

> [!info]- 题目描述 (点击展开)
> 给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的 **异位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。
> 
> **异位词** 指由相同字母重排列形成的字符串（包括相同的字符串）。
> 
> **示例 1:**
> 输入: s = "cbaebabacd", p = "abc"
> 输出: [0,6]
> 解释:
> 起始索引等于 0 的子串是 "cba", 它是 "abc" 的异位词。
> 起始索引等于 6 的子串是 "bac", 它是 "abc" 的异位词。
> 
> **示例 2:**
> 输入: s = "abab", p = "ab"
> 输出: [0,1,2]
> 
> **提示:**
> * $1 <= s.length, p.length <= 3 * 10^4$
> * `s` 和 `p` 仅包含小写字母

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略
1.  **数据结构优化：** 题目仅包含小写字母，放弃重量级的 `HashMap`，改用两个长度为 26 的 `int` 数组 (`pCount`, `sCount`) 来统计字符频率。数组下标 `0-25` 对应 `a-z`。
2.  **固定窗口滑动：** 维护一个长度固定为 `p.length()` 的滑动窗口。
    * **初始化：** 先统计 `p` 的字符，并把窗口在 `s` 中前 `p.length()` 个字符统计好。
    * **滑动过程：** 每次向右移动一格：
        * **Check：** 比较两个数组内容是否完全一致 (`Arrays.equals`)。
        * **Slide：** 窗口右移 -> 计数器 `sCount` 中 **移除** 左边出去的字符，**添加** 右边进来的字符。

### 为什么选择这个解法？
* **性能极致：** `Arrays.equals` 比较两个长度为 26 的数组非常快（视为 $O(1)$），且数组索引访问没有哈希冲突，优于 HashMap。

## 💻 代码实现 (Java)

```java
import java.util.*;

class Solution {
    public List<Integer> findAnagrams(String s, String p) {
        List<Integer> list = new ArrayList<>();
        
        // 1. 边界条件处理
        if (s.length() < p.length()) {
            return list;
        }

        // 2. 定义两个数组代替 HashMap (a-z 对应索引 0-25)
        int[] pMap = new int[26];
        int[] sMap = new int[26];

        // 3. 初始化：统计 p 的字符 和 s 中第一个窗口的字符
        for (int i = 0; i < p.length(); i++) {
            pMap[p.charAt(i) - 'a']++;
            sMap[s.charAt(i) - 'a']++;
        }

        // 4. 开始滑动窗口
        // i 代表当前窗口的【起始位置】
        // 循环条件解释：s.length() - p.length() 是最后一个可能的起始位置
        for (int i = 0; i <= s.length() - p.length(); i++) {
            
            // Step 1: 检查当前窗口是否匹配
            if (Arrays.equals(sMap, pMap)) {
                list.add(i);
            }

            // Step 2: 窗口右移 (Slide)
            // 只有当窗口还没滑到最后时，才进行更新操作
            if (i < s.length() - p.length()) {
                // 移除左边的字符 (当前 i 位置)
                sMap[s.charAt(i) - 'a']--;
                // 加入右边的字符 (窗口右侧的下一个位置 i + p.length())
                sMap[s.charAt(i + p.length()) - 'a']++;
            }
        }

        return list;
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(N)$
  - *分析：* 其中 $N$ 是字符串 `s` 的长度。虽然循环内部调用了 `Arrays.equals`，但数组长度固定为 26，所以可以视为常数操作 $O(1)$。整体只需要遍历一次字符串。
- **空间复杂度：** $O(1)$
  - *分析：* 我们只使用了两个长度为 26 的整型数组，不管字符串多长，额外空间都是固定的。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **Map 逻辑陷阱**：最初使用 `HashMap` 时，不仅效率低，还容易踩坑（比如计数减为 0 后 Key 还在，导致 `equals` 判为 false）。**数组是这道题的终极解法。**
> > 2. **包装类开销**：在循环里写 `Integer now = ...` 会产生大量的自动装箱对象，不仅慢还占内存。直接用 `int` 和 `char` 运算才是正解。
> > 3. **边界判断**：滑动窗口最容易写错的就是循环结束条件，以及右边元素加入时是否越界。

> [!tip] 关键技巧 / 总结
> > 1. **异位词 = 词频统计**：看到“异位词”、“排列”这类字眼，立刻想到统计每个字母出现的次数。
> > 2. **定长窗口**：因为异位词长度一定等于 `p.length()`，所以窗口大小是固定的，这比变长窗口（LeetCode 3题）逻辑更简单：一进一出即可。

## 🧠 深度思考与感悟 (Reflections)
- **💡 触类旁通：** 这种 `int[26]` 的技巧不仅用于“异位词”，还适用于“判断字符串是否包含排列”、“最小覆盖子串(特定字符集)”等所有涉及**字符计数**的问题。
- **🔧 工程思维：** 在处理 ASCII 字符集时，数组永远是比 Map 更轻量级的选择。这提醒我们在写业务代码时，如果 Key 的范围非常小且连续（比如月份 1-12，星期 1-7），数组往往优于 Map。
- **✨ 瞬间感悟：** 所谓的“滑动窗口”，其实就像是一列火车在轨道上跑，车厢长度不变（`p.length`），车头吃进来一个数据，车尾就要吐出去一个数据，永远保持平衡！
