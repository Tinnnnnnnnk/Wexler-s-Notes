# 394. 字符串解码 (Decode String)

> [!info]- 题目描述 (点击展开)
> > 给定一个经过编码的字符串，返回它解码后的字符串。
> > 编码规则为: `k[encoded_string]`，表示其中方括号内部的 `encoded_string` 正好重复 `k` 次。注意 `k` 保证为正整数。
> > **示例 1：** 输入：`s = "3[a]2[bc]"`  输出：`"aaabcbc"`
> > **示例 2：** 输入：`s = "3[a2[c]]"` 输出：`"accaccacc"` (嵌套括号)

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略 (工作台与保险柜模式 / 双栈法)
彻底抛弃普通的字符拼接，引入**“上下文切换 (Context Switching)”**思想。维护两个栈（保险柜）和一个当前工作台。
1.  **第一步 (拆分职责)：** 建立数字栈 `numStack` 记录倍数，建立字符串栈 `strStack` 记录外层未完成的底座。建立 `currentStr` 和 `k` 作为当前工作台。
2.  **第二步 (处理数字与字母)：** 遇到数字，更新当前倍数 `k = k * 10 + (c - '0')`（防多位数）；遇到字母，直接拼入 `currentStr`。
3.  **第三步 (遇到 `[` 存档)：** 遇到嵌套大门，把工作台上的 `k` 和 `currentStr` 锁进各自的栈中保存现场。然后**清空工作台**，准备处理内层。
4.  **第四步 (遇到 `]` 读档)：** 内层处理完后，从栈中掏出外层的“倍数”和“底座”。将内层结果按倍数放大后，**拼接到外层底座的后面**。

### 为什么选择这个解法？
利用栈“后进先出”的物理特性，完美化解了**俄罗斯套娃（递归嵌套）**的问题。无论括号嵌套多少层，空间状态都能被精准封存和恢复，避免了极其复杂的正则表达式或递归的压栈开销。

## 💻 代码实现 (Java)

```java
import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public String decodeString(String s) {
        Deque<Integer> numStack = new ArrayDeque<>();
        Deque<StringBuilder> strStack = new ArrayDeque<>();
        
        StringBuilder currentStr = new StringBuilder();
        int k = 0;

        for (char c : s.toCharArray()) {
            if (Character.isDigit(c)) {
                // 细节：处理 100 这种多位数，利用 ASCII 差值转真实数字
                k = k * 10 + (c - '0');
            } else if (c == '[') {
                // 存档现场：进入内层前，把外层的状态压栈
                numStack.push(k);
                strStack.push(currentStr);
                // 清空工作台，开干内层
                currentStr = new StringBuilder();
                k = 0;
            } else if (c == ']') {
                // 读档合并：内层干完了，拿外层的数据出来组装
                StringBuilder temp = currentStr; // 内层拼好的小积木
                currentStr = strStack.pop();     // 外层的旧底座
                int repeatTimes = numStack.pop(); // 外层要求的倍数
                
                // 疯狂拼接
                for (int i = 0; i < repeatTimes; i++) {
                    currentStr.append(temp);
                }
            } else {
                // 普通字母，直接在工作台干活
                currentStr.append(c);
            }
        }
        return currentStr.toString();
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(N)$ (严格来说是 $O(\text{解码后的字符串长度})$)
  - *分析：* 每个字符都会被遍历一次。拼接操作的时间取决于最终生成的字符串总长度。总体上是极其高效的线性时间。
- **空间复杂度：** $O(M)$ (其中 $M$ 是括号的嵌套层数)
  - *分析：* 使用了两个额外的栈。在最坏情况下（如 `3[2[1[a]]]`），栈的深度等于左括号的数量。空间开销极小。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **多位数陷阱：** 遇到数字不能只查一次！`100` 不是拆开的三个数字，必须用 `k = k * 10 + num` 累加组合。
> > 2. **强转 ASCII 陷阱：** 如果栈里存的是 `char`，直接 `int k = char` 拿到的是 ASCII 码（比如 `'3'` 变成 51）。必须用 `c - '0'`。
> > 3. **套娃组装陷阱：** 从内层退出来时，算好的字符串**不能直接怼到全局答案里**，而是必须拼在**上一层传下来的底座**后面！

> [!tip] 关键技巧 / 总结
> > **永远不要在 Java 里用 `java.util.Stack`！** 它底层加了沉重的线程锁。任何需要用到栈的场景，无脑使用 `Deque<XXX> stack = new ArrayDeque<>();`，速度拉满！

## 🧠 深度思考与感悟 (Reflections)
- **💡  触类旁通：** 这种“遇左括号压栈，遇右括号弹栈结算”的打法，其实就是经典的**“状态机”**。以后在写诸如 `2 * (3 + 4)` 这种**计算器解析引擎 (AST 构建)**时，底层用的也是一模一样的双栈法！
- **🔧 工程思维：** 代码的可读性胜过一切。双栈法虽然多开了一个栈，但它把数字和字符串的逻辑完全解耦了，这比维护一个能存万物的泛型栈要优雅、清晰一万倍。
- **✨ 瞬间感悟：** 栈存在的终极意义，不仅是“逆序”，更是**“时间冻结与时空穿梭（保存现场与恢复现场）”**！
