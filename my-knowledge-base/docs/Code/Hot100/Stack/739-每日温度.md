# 739. 每日温度 (Daily Temperatures)

> [!info]- 题目描述 (点击展开)
> > 给定一个整数数组 `temperatures` ，表示每天的温度，返回一个数组 `answer` ，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。
> > 如果气温在这之后都不会升高，请在该位置用 `0` 来代替。
> > **示例：** 输入: `temperatures = [73,74,75,71,69,72,76,73]`，输出: `[1,1,4,2,1,1,0,0]`

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略 (单调递减栈)
寻找“下一个更大元素”的绝对标准打法。核心要义是：**栈内永远只存每一天的“索引 (Index)”，绝不存具体的“温度值”！**

1.  **第一步 (初始化)：** 准备一个 `days` 数组存放结果（Java 默认全为 0，省去后续清零），准备一个 `Deque<Integer>` 作为单调栈。
2.  **第二步 (遍历与查岗)：** 每天拿着当天的温度 `temperatures[i]` 去问栈顶的老哥（之前的日子）。如果今天的温度比栈顶老哥 `deque.peek()` 记录那天的温度还要高，说明栈顶老哥终于等到了他要的“升温日”！
3.  **第三步 (结算与出栈)：** 把栈顶老哥的索引 `prevIndex` 弹出来。他等了几天？极其简单：用今天的索引减去他的索引，即 `days[prevIndex] = i - prevIndex`。继续用 while 循环查下一个栈顶，直到栈空或者今天的温度不再比栈顶高。
4.  **第四步 (入栈等待)：** 查岗完毕后，把今天的索引 `i` 压入栈中，等待未来某一天来把它结算掉。

### 为什么选择这个解法？
如果用双层 for 循环暴力向后找，时间复杂度是 $O(N^2)$，面对海量数据必定超时。单调栈让每一个元素**最多入栈一次、出栈一次**，将时间复杂度进行了降维打击，压到了极致的 $O(N)$。

## 💻 代码实现 (Java)

```java
import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] days = new int[n]; // Java 默认初始化为 0，无需最后再清零
        Deque<Integer> deque = new ArrayDeque<>(); // 存储的是“索引”，不是温度值！
        
        for (int i = 0; i < n; i++) {
            // 当栈不为空，且今天的温度 > 栈顶索引所对应的温度时，开始结算
            while (!deque.isEmpty() && temperatures[i] > temperatures[deque.peek()]) {
                int prevIndex = deque.pop(); // 弹出那个等到了升温的“倒霉蛋”的索引
                days[prevIndex] = i - prevIndex; // 坐标相减，直接得出距离天数
            }
            // 结算完后，把今天的索引也乖乖放进栈里，等后面的人来捞
            deque.push(i);
        }
        
        return days;
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(N)$
  - *分析：* 数组中每个索引 `i` 最多被 `push` 进栈一次，也最多被 `pop` 出栈一次。虽然有一个 `while` 循环，但全过程的操作总数是 $2N$，摊还下来仍然是严格的 $O(N)$ 线性时间。
- **空间复杂度：** $O(N)$
  - *分析：* 极限情况下（例如气温每天都在持续下降 `[100, 90, 80, 70]`），所有的索引都会被压进栈里无法结算，此时栈的大小达到了 $N$。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **循环条件粗心：** `for (int i=0; i>length; i++)` 直接导致循环一次都不进。必须是 `i < length`。
> > 2. **画蛇添足的自增变量：** 曾试图在 while 里用一个额外的变量 `int t = 1; t++;` 来盲猜流逝的天数。遇到不连续的弹出时会导致极其严重的计算错乱。
> > 3. **多余的清理工作：** 在结尾用 for 循环把没结算的日子填 0。Java 里的 `new int[]` 默认就是 0，直接略过。

> [!tip] 关键技巧 / 总结
> > **永远存索引！永远存索引！永远存索引！**
> > 在单调栈里，只要题目求的是“距离”、“跨度”、“宽度”，栈里塞的一定是坐标 `i`。因为拿到了坐标，不但能算距离，还能用 `arr[i]` 随时查值；但如果只存值，就彻底丢失了时间维度。
