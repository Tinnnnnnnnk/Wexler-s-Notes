# 560. 和为 K 的子数组

> [!info]- 题目描述 (点击展开)
> 给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回 *该数组中和为 `k` 的子数组的个数* 。
> 
> **子数组** 是数组中元素的连续非空序列。
> 
> **示例 1：**
> 输入：nums = [1,1,1], k = 2
> 输出：2
> 
> **示例 2：**
> 输入：nums = [1,2,3], k = 3
> 输出：2
> 
> **提示：**
> * $1 <= nums.length <= 2 * 10^4$
> * $-1000 <= nums[i] <= 1000$
> * $-10^7 <= k <= 10^7$

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略：前缀和 + 哈希表 (两数之和进阶版)
1.  **数学转换：**
    * 原问题是寻找子数组 `nums[i..j]` 使得和为 `k`。
    * 利用前缀和公式转化：$PreSum[j] - PreSum[i-1] = k$。
    * 移项得到查找目标：**$PreSum[i-1] = PreSum[j] - k$**。
2.  **算法流程：**
    * 维护一个变量 `preSum` 记录当前累加和。
    * 维护一个 `HashMap` 记录 **{前缀和 : 出现次数}**。
    * 遍历数组，对于每个 `preSum`，先去 Map 里查是否存在 `preSum - k`。如果有，说明之前有几个位置可以作为子数组的起点，直接累加次数。
    * 最后将当前的 `preSum` 存入 Map。

### 为什么选择这个解法？(对比分析)
* **我的原解法 (暴力前缀和)：** * 虽然想到了前缀和数组 `keys[]`，但使用了**双重循环**来枚举所有可能的起止点 $(i, j)$。
    * **缺点**：时间复杂度是 $O(N^2)$。当 $N=20000$ 时，计算量高达 $4 \times 10^8$，会导致 **TLE (超时)**。
* **优化后解法 (哈希表)：**
    * 利用“两数之和”的思路，通过空间换时间。
    * **优点**：只需要遍历一次数组，查找 Map 是 $O(1)$ 的操作。总时间复杂度降低为 **$O(N)$**，这在工程和面试中是质的飞跃。

## 💻 代码实现 (Java)

```java
import java.util.HashMap;

class Solution {
    public int subarraySum(int[] nums, int k) {
        // Key: 前缀和, Value: 该前缀和出现的次数
        HashMap<Integer, Integer> map = new HashMap<>();
        
        // 🌟 核心细节：初始化 (0, 1)
        // 含义：前缀和为 0 的情况默认出现过 1 次。
        // 作用：如果某个子数组 nums[0...i] 本身的和就是 k，
        // 那么 preSum - k = 0，此时需要 map 里有 0 才能统计到这种情况。
        map.put(0, 1);
        
        int count = 0;
        int preSum = 0;
        
        for (int x : nums) {
            // 1. 更新当前前缀和
            preSum += x;
            
            // 2. 查账本：看看历史记录里有没有 (preSum - k)
            if (map.containsKey(preSum - k)) {
                count += map.get(preSum - k);
            }
            
            // 3. 记账：把当前的前缀和记录下来
            // getOrDefault 避免了空指针判断，简洁优雅
            map.put(preSum, map.getOrDefault(preSum, 0) + 1);
        }
        
        return count;
    }
}
```

## 📊 复杂度分析
- **时间复杂度：** $O(N)$
  - *分析：* 我们只对数组进行了一次遍历。`HashMap` 的插入和查询操作平均时间复杂度为 $O(1)$。相比于暴力法的 $O(N^2)$，这是巨大的提升。
- **空间复杂度：** $O(N)$
  - *分析：* 最坏情况下（例如数组所有元素为正数），所有的前缀和都不相同，`HashMap` 需要存储 $N$ 个键值对。

## 📝 错题本 / 复盘
> [!failure] 遇到的坑 / 错误点
> > 1. **漏掉初始化**：忘记写 `map.put(0, 1)`。这会导致从数组开头 `nums[0...i]` 就满足条件的子数组被漏算。
> > 2. **顺序问题**：在循环体内，应该是 **先查询 (Check)**，**后更新 (Put)**。如果先 Put 再 Check，当 $k=0$ 时，会把当前这个空集也算进去（或者重复计算），逻辑上不严谨。

> [!tip] 关键技巧 / 总结
> > 1. **前缀和 + 哈希表** 是解决“连续子数组和”问题的黄金搭档。
> > 2. `map.getOrDefault(key, 0) + 1` 是统计频率的标准写法，一定要熟练掌握。

## 🧠 深度思考与感悟 (Reflections)
- **💡 触类旁通：** 这道题其实就是 **LeetCode 1. 两数之和** 的变种！
    * 两数之和：$nums[i] + nums[j] = target \rightarrow nums[i] = target - nums[j]$
    * 本题：$PreSum[j] - PreSum[i] = k \rightarrow PreSum[i] = PreSum[j] - k$
    * 本质都是：**固定一个数，去哈希表里找另一个数是否存在。**
- **🔧 工程思维：** **空间换时间** (Space-Time Tradeoff)。我们牺牲了 $O(N)$ 的内存来存储历史状态，换取了从 $O(N^2)$ 到 $O(N)$ 的速度提升。在海量数据处理中，这种交换通常是非常划算的。
- **✨ 瞬间感悟：** 不要总是想着“往回找”（双指针/暴力），试试“往前看”并把走过的路记在小本本（Map）上，需要的时候回头查一下本子就行了！
思路是依此遍历nums数组，每一次都获取前i个数的前缀和；我们需要其中一截子数组b **（我们假设前缀和有两截数组，不需要的是a，需要的是b）**，从某一个节点开始一直到当前的节点；这个b的和是k，也就是说k=总的前缀和-a；而a可以在我们每一次遍历中逐步添加，我们每遍历一次就往HashMap中添加一个数据，等我们需要的时候，就可以直接根据a的个数求得子串的个数
