# 15. 三数之和 (3Sum)

> [!info]- 题目描述 (点击展开)
> 给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。
> 
> **注意：** 答案中不可以包含重复的三元组。
> 
> **示例 1：**
> 输入：`nums = [-1,0,1,2,-1,-4]`
> 输出：`[[-1,-1,2],[-1,0,1]]`
> 
> **提示：**
> - `3 <= nums.length <= 3000`
> - `-105 <= nums[i] <= 105`

---

## 💡 解题思路 (Kasumi's Memo)

### 核心策略：排序 + 双指针
这道题暴力的复杂度是 $O(N^3)$，绝对会超时。利用**双指针**可以将复杂度降为 $O(N^2)$。

1.  **预处理：** 先对数组进行**排序**。这是双指针能生效的前提（利用单调性）。
2.  **固定主唱 (Key)：** 遍历数组，固定 `nums[i]` 为第一个数。
3.  **双指针伴奏：** -   `left = i + 1` (从 Key 的后一位开始)
    -   `right = length - 1` (数组末尾)
4.  **寻找合奏 (Sum)：** 计算 `sum = nums[i] + nums[left] + nums[right]`
    -   若 `sum > 0`：数大了 -> `right` 左移。
    -   若 `sum < 0`：数小了 -> `left` 右移。
    -   若 `sum == 0`：**记录结果**，并同时收缩左右指针。

### 🚨 关键难点：如何优雅地去重？
这道题最容易出错的地方就是**重复解**。
1.  **Key 的去重：** 每一轮 `i` 循环开始时，如果 `nums[i] == nums[i-1]`，说明这个数刚才已经当过主唱了，**直接 continue**。
2.  **左右指针的去重：** 只有在 `sum == 0` (找到解) 的时候才去重。
    -   如果 `nums[left] == nums[left+1]`，`left` 继续右滑。
    -   如果 `nums[right] == nums[right-1]`，`right` 继续左滑。

## 💻 优化代码 (Java)

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums); // 必须排序

        for (int i = 0; i < nums.length; i++) {
            // 剪枝：如果当前已经大于0，后面不可能凑出0了
            if (nums[i] > 0) break; 
            
            // 去重1：跳过重复的 Key
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            int left = i + 1;
            int right = nums.length - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    
                    // 去重2：找到答案后，跳过重复的元素
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    
                    left++;
                    right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return result;
    }
}
```

## 🧠 深度思考与感悟 (Reflections)
-  **💡  触类旁通：** 
- **🔧 工程思维：** 
- **✨ 瞬间感悟：** 
	- 最开始的想法是使用哈希表来先将每个数存在Hash表中，自带排序的那种；遍历每 一个Hash表，选择一个元素；选择的数字假如是a，我们需要找到两个加起来为-a的；从头遍历，判断遍历到的数字加上-a得到的数字是否在Hash表中存在；如果存在就将这三个数字转为数组存到List<List< Integer>>中
	- 不可否认，这种暴力法可以，但是会大大超出运行时间，而且在双指针模块下却没有使用到双指针，明显是不对的，因此转换思路
	 - 首先给数组进行排序操作，排序完之后再遍历数组并使用双指针操作；由于数组有序，我们可以直接将左右指针都设置在被第一次遍历的元素的一侧，防止左右指针遍历到这个第一次被遍历到的元素；在遍历时一定要注意，当情况符合题意时，注意一下往后遍历的规则
	 - **多关注剪枝操作**，多思考目前是否可以剪枝
