# Day 2｜“案底查询”：一次上传在 MySQL 留下什么，在 Redis 留下什么 🧾

## 今日目标

---

## 具体怎么做

### Step 1：找到“写入 DB 的地方”（不要猜表名）

在入口 Controller 往下走，看到类似：

- `fileRepository.save(...)`
    
- `fileMapper.insert(...)`
    
- `FileInfoService.create(...)`
    

✅ 你做两件事：

- 在 `save/insert` 这一行打断点
    
- 断点停住时，把“即将写入对象”的字段展开看一遍，重点记：
    
    - `md5`
        
    - `url/objectKey`
        
    - `status`（比如 UPLOADING/PROCESSING/FINISHED）
        
    - `orgTag` / `creatorId`（权限会用）
        

> 证据：`evidence/day02/01_db_entity_fields.png`

### Step 2：用 mysql 命令行查最新一条记录（不用额外客户端）

进入 MySQL 容器或本机 MySQL 执行（示例）：

```bash
# 进入 mysql（如果是 docker）
docker exec -it <mysql容器名> mysql -u<user> -p

# 选择库
use <你的数据库名>;

# 列表看看有哪些文件表（别猜）
show tables;

# 找到明显的文件表后，查最新一条
select * from <文件表名> order by id desc limit 1;
```

✅ 你要对上：

- DB 里的 `url/objectKey` 是否能在 MinIO 控制台找到对应对象
    
- DB 里的 `md5` 是否与入口断点一致
    

> 证据：`evidence/day02/02_mysql_latest_row.txt`

### Step 3：查 Redis 是否有上传进度相关 key（用 redis-cli）

```bash
docker exec -it <redis容器名> redis-cli

# 不用 KEYS（坏习惯），用 SCAN
SCAN 0 MATCH *upload* COUNT 100
SCAN 0 MATCH *<你刚才看到的md5片段>* COUNT 100

# 找到可疑 key 后：
TYPE <key>
```

如果是 bitmap，你通常会看到它是 `string` 类型，但可以用 bit 命令操作：

```bash
GETBIT <key> 0
BITCOUNT <key>
STRLEN <key>
```

> 证据：`evidence/day02/03_redis_keys_and_type.txt`

2. **上传单个分片**（会写 MinIO 分片对象 + 更新 Redis bit）
    

### Step 2：在“写 Redis bit”的那一行打断点

```bash

```

按 F8 执行 setbit 后再查一次：

```bash
GETBIT <key> <chunkIndex>
BITCOUNT <key>
```

✅ 你要看到：

- `GETBIT` 从 0 变 1
    
- `BITCOUNT` 增加 1（或符合预期）
    

> 证据：`evidence/day03/02_redis_bit_before_after.txt`
