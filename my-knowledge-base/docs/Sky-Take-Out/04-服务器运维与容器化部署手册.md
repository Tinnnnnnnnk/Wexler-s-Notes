## 1. Linux 系统基础

### 1.1 系统状态与电源管理
* **`init 0`**：关闭系统 (Shutdown)。
* **`init 6`**：重启系统 (Reboot)。

### 1.2 Shell 特殊符号
在命令行交互中，熟练使用特殊符号可以极大提高效率。

* **管道符 (`|`)**：
    * **作用**：将前一个命令的输出结果，作为后一个命令的输入参数。
    * **示例**：`ps -ef | grep java` (查看所有进程，并过滤出包含 java 关键字的行)。

* **输出重定向 (`>`)**：
    * **作用**：将命令的输出写入到指定文件中。
    * **特性**：**覆盖**写入。如果文件不存在则创建，如果存在则清空原内容后写入。
    * **示例**：`echo "Hello" > test.txt`

* **追加重定向 (`>>`)**：
    * **作用**：将命令的输出追加到指定文件末尾。
    * **特性**：**追加**写入。不会清空原文件内容。
    * **示例**：`cat info.log >> error.log`

## 2. Docker 容器化技术

### 2.1 核心概念
* **镜像 (Image)**：
    * Docker 镜像是一个只读的模板。
    * 例如：一个镜像可以包含一个完整的 Ubuntu 操作系统环境，里面仅安装了 Apache 或用户需要的其他应用程序。
    * 镜像可以用来创建 Docker 容器。
* **容器 (Container)**：
    * Docker 利用容器来运行应用。
    * 容器是从镜像创建的运行实例。它可以被启动、开始、停止、删除。每个容器都是相互隔离的、保证安全的平台。
* **仓库 (Repository)**：
    * 集中存放镜像文件的场所。
    * 最大的公开仓库是 Docker Hub。

### 2.2 Docker 指令生命周期

Docker 的操作主要围绕镜像与容器的生命周期展开。



#### (1) 镜像操作 (Image)
* **`docker build`**：
    * **作用**：从 Dockerfile 构建镜像。
    * **语法**：`docker build -t image_name:tag .` (注意最后的点，代表当前目录)。
* **`docker pull`**：
    * **作用**：从远程仓库拉取镜像。
    * **语法**：`docker pull image_name:tag` (如果不写 tag，默认 latest)。
* **`docker push`**：
    * **作用**：将本地镜像推送到远程仓库。
    * **语法**：`docker push image_name:tag`。
* **`docker images`**：
    * **作用**：列出本地所有镜像。
* **`docker rmi`**：
    * **作用**：删除本地镜像。
    * **语法**：`docker rmi image_id`。

#### (2) 容器操作 (Container)
* **`docker run`**：
    * **作用**：创建一个新容器并运行。
    * **语法**：`docker run [options] image_name [command]`
    * **常用参数**：
        * `-d`: 后台运行容器 (Detached mode)。
        * `-p`: 端口映射 (宿主机端口:容器端口)。例如 `-p 8080:80`。
        * `--name`: 给容器起个名字。
        * `-v`: 挂载数据卷 (Volume)。
* **`docker ps`**：
    * **作用**：查看当前正在运行的容器。
    * **参数**：`-a` 查看所有容器（包括已停止的）。
* **`docker stop`**：
    * **作用**：停止一个运行中的容器。
* **`docker start`**：
    * **作用**：启动一个已停止的容器。
* **`docker restart`**：
    * **作用**：重启容器。
* **`docker rm`**：
    * **作用**：删除容器。
    * **注意**：只能删除已停止的容器。如果容器正在运行，需添加 `-f` 强制删除。

#### (3) 运维与排查
* **`docker logs`**：
    * **作用**：查看容器日志。
    * **语法**：`docker logs -f container_id` (`-f` 表示 follow，实时跟踪)。
* **`docker exec`**：
    * **作用**：在运行的容器中执行命令。
    * **常用**：`docker exec -it container_id /bin/bash` (进入容器的终端进行交互)。

#### (4) 镜像迁移 (离线部署)
* **`docker save`**：
    * **作用**：将镜像保存为 tar 归档文件。
    * **语法**：`docker save -o my_image.tar image_name`。
* **`docker load`**：
    * **作用**：从 tar 归档文件加载镜像。
    * **语法**：`docker load -i my_image.tar`。

---

> [!tip] 常见问题排查
> * **端口冲突**：如果 `docker run` 报错端口被占用，检查宿主机端口是否被其他进程（如本地的 Tomcat 或 Nginx）占用。
> * **权限问题**：执行 docker 命令如果提示 `permission denied`，尝试在命令前加 `sudo`，或者将当前用户加入 docker 用户组。
> * **资源清理**：长期运行后，会有很多 `<none>` 的悬空镜像，可以使用 `docker image prune` 清理。
