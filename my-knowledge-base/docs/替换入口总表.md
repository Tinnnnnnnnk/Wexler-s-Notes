# 替换入口总表（Next.js 当前架构版）

> 本文档基于当前项目实际代码结构整理（`wexler-notes-next`），用于快速替换首页背景视频、背景音乐和相关视觉配置。

- 前端主项目目录：`D:\Github\Wexler-s-Notes\wexler-notes-next`
- 首页背景素材目录：`D:\Github\Wexler-s-Notes\wexler-notes-next\public\media\home-bg`
- 首页 BGM 素材目录：`D:\Github\Wexler-s-Notes\wexler-notes-next\public\media\home-bgm`

## 1. 背景视频 / 背景图替换入口

### 1.1 素材放置位置

- 视频与背景图都放到：
  - `wexler-notes-next/public/media/home-bg/`

### 1.2 代码配置入口（真正生效点）

- 文件：`wexler-notes-next/src/components/home/Backdrop.tsx`
- 关键常量：
  - `IMAGE_CANDIDATES`
  - `VIDEO_CANDIDATES`

当前逻辑：
1. 默认优先走 `VIDEO_CANDIDATES`（只要视频能加载）。
2. 如果视频候选全部失败，自动回退到 `IMAGE_CANDIDATES[0]`。
3. `VIDEO_CANDIDATES` 支持多候选，按数组顺序自动回退。

### 1.3 视觉强度配置

- 文件：`wexler-notes-next/src/components/home/Backdrop.module.css`
- 可调项：
  - `.liquid::after`（液态叠层强度）
  - `.aura`（液态氛围层）
  - `.lowFps::after`（低帧率时的降载样式）

## 2. 液态首页 BGM 替换入口

### 2.1 素材放置位置

- 音频文件放到：
  - `wexler-notes-next/public/media/home-bgm/`

### 2.2 代码配置入口（真正生效点）

- 文件：`wexler-notes-next/src/components/home/BgmPlayer.tsx`
- 关键常量：
  - `BGM_SOURCE_TEMPLATES`
  - `BGM_TITLE`
  - `BGM_ARTIST`

当前音频回退顺序（从上到下）：
1. `/media/home-bgm/liquid-bgm.ogg`
2. `/media/home-bgm/liquid-bgm.opus`
3. `/media/home-bgm/liquid-bgm.mp3`
4. `/media/home-bgm/liquid-bgm.flac`

说明：
1. 浏览器会按 `<source>` 顺序尝试可解码格式。
2. 组件包含 `detectRuntimeBasePath()`，部署在子路径时会自动拼接前缀路径。

### 2.3 播放器样式配置

- 文件：`wexler-notes-next/src/components/home/BgmPlayer.module.css`
- 可调项：
  - `.player` / `.stagePlayer`（透明度、模糊、阴影、尺寸）

## 5. 快速替换模板（推荐）

### 5.1 替换背景视频

1. 把新视频复制到 `public/media/home-bg/`
2. 打开 `Backdrop.tsx`，改 `VIDEO_CANDIDATES`
3. 把你希望优先播放的视频放在数组第一个

### 5.2 替换背景音乐

1. 把新音频复制到 `public/media/home-bgm/`
2. 打开 `BgmPlayer.tsx`，改 `BGM_SOURCE_TEMPLATES`
3. 把兼容性最高格式（一般 mp3 或 ogg）放前面
4. 同步更新 `BGM_TITLE` / `BGM_ARTIST`

---

## 6. 替换后自检清单

1. 本地启动：
   - `npm --prefix wexler-notes-next run dev`
2. 浏览器直接访问素材 URL（确保 200）：
   - `http://localhost:3000/media/home-bg/你的文件名.mp4`
   - `http://localhost:3000/media/home-bgm/你的文件名.ogg`（或 mp3）
3. 首页切换到液态模式，验证：
   - 背景视频是否播放
   - 播放器是否能播放/暂停/拖动
   - 控制台是否报 404 或解码错误

## 7. 常见误区（当前项目）

1. 旧文档中的 `HomeFxBackdrop.vue`、`style.css`、`homeFxState.js` 已不是当前生效入口（这是旧架构内容）。

3. 文件放进 `my-knowledge-base/docs/public/...` 不会影响 Next 首页，Next 首页读取的是 `wexler-notes-next/public/...`。
