### 线程和进程的区别是什么
- 进程是操作系统分配资源的最小单位
- 线程是进程中的独立执行单元，多个线程共享同一个进程的资源
### 协程是什么
- 协程是比线程更轻量级的并发单元，可以在单线程中实现并发执行
- 协程在用户态进行调度，避免了线程切换的内核态转换开销
- JAVA本身不支持协程，可以使用Quasar、Kotlin等框架来实现
### 线程之间如何通信
- Java采用共享内存的并发模型，JMM
### 线程有哪些创建方式
- 重写父类Thread 的 run( )方法
	- ```
	  class ThreadTask extends Thread {
    public void run() {
        System.out.println("看完二哥的 Java 进阶之路，上岸了!");
    }

    public static void main(String[] args) {
        ThreadTask task = new ThreadTask();
        task.start();
    }
}
	  ```
	- 调用 start( )方法来启动线程
	- 如果ThreadTask 继承了另外一个类，那么就不能再继承 Thread 类，因为JAVA 不支持多重继承
- 重写Runnable接口的 run( ) 方法
	- ```
	  class RunnableTask implements Runnable {
    public void run() {
        System.out.println("看完二哥的 Java 进阶之路，上岸了!");
    }

    public static void main(String[] args) {
        RunnableTask task = new RunnableTask();
        Thread thread = new Thread(task);
        thread.start();
    }
}
	  ```
	- 将实现类的对象作为参数传递给Thread对象的构造方法，最后调用start( )方法启动线程
	- 避免了JAVA的单继承限制
- 重写Callable接口的 call( )方法
	- ```
	  class CallableTask implements Callable<String> {
    public String call() {
        return "看完二哥的 Java 进阶之路，上岸了!";
    }

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        CallableTask task = new CallableTask();
        FutureTask<String> futureTask = new FutureTask<>(task);
        Thread thread = new Thread(futureTask);
        thread.start();
        System.out.println(futureTask.get());
    }
}
	  ```
	- 创建FutureTask对象，参数为Callable实现类的对象，紧接着创建Thread对象，参数为FutureTask对象，最后调用start( )方法启动线程
	- 可以获取线程执行的结果

### 调用 start 方法会执行 run方法，为什么不直接调用run 方法
- 调用 start 方法会创建一个新的线程，并异步执行 run方法中的代码
- 如果直接调用了 run 方法，那么只会在当前的主线程中执行run 方法中的所有代码，没有创建新的线程，也就达不到多线程并发的目的
### 线程的常用调度方法
- stop方法
	- stop方法用来强制停止线程，可能会在不一致的状态下释放锁，因此被废弃了
- interrupt方法
	- interrupt方法用来通知线程停止，但不会直接停止，会让线程自己去停止
