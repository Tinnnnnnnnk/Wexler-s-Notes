### 怎么理解面向对象？封装继承多态是什么？
- 面向对象就是把现实事物抽象为对象，对象具有属性和行为。
- 封装：把事物的属性和行为结合起来，对外隐藏对象的属性，只保留对象提供的接口供外部交互
- 继承：是一种子类自动共享父类数据结构和方法的机制，是一种代码复用手段
- 多态：是指允许不同类的对象对同意消息做出响应，也就是同一接口，使用不同的实例来执行不同的操作
#### 多态的体现
- 方法重载：同一个类中可以有多个同名方法，各自拥有不同的参数列表，根据传入的参数，编译器会决定使用哪一个方法，例如 `add(int a, int b)` 和 `add(double a, double b)`
- 方法重写：在一个父类中定义一个`Eat` 方法，可以分别在子类`Cat`和`Dog`中分别重写这个方法来实现不一样的操作
- 接口与实现：多个类可以实现同一个接口
- 向上转型和向下转型
	- 向上转型是：**把子类对象赋值给父类类型的引用变量**。
	- ```
	  // 这就是向上转型！把 Guitar 对象，当作 Instrument 来对待。 
	  Instrument myInstrument = new Guitar();
	  ```
	- 也就是 可以使用父类类型的引用指向子类对象
	- 向下转型是： **将父类引用转回其子类类型**
	- ```
	  // 向下转型！把泛指的乐器，强行变回吉他
	  Guitar myGuitar = (Guitar) myInstrument; 
      // 可以用吉他专属的技能
      myGuitar.useWhammyBar();
	  ```
#### 多态的条件
- 子类继承父类
- 子类重写父类的方法
- 父类引用指向子类对象
#### 多态的实现原理是什么
- 多态通过动态绑定实现，Java 使用虚方法表存储方法指针，方法调用时根据对象实际类型从虚方法表查找具体实现。
### 重载和重写的区别
- 如果一个类有多个名字相同但参数个数不同的方法，我们通常称这些方法为方法重载。
- 如果子类具有和父类一样的方法（参数相同、返回类型相同、方法名相同，但方法体不同），我们称之为方法重写。
- 重写是父类和子类中的东西，重载是一个类中的多个同名方法

### 抽象类与接口
#### 抽象类
```
// 这是一个抽象类（半成品模板），不能直接 new 一个 出来哦！
public abstract class BandMember {
    
    String name; // 普通属性：大家都有名字

    public BandMember(String name) {
        this.name = name;
    }

    // 1. 普通方法（已经写好的）：所有成员排练完都会喊累，做法完全一样，直接在模板里写死！
    public void complain() {
        System.out.println(name + " 说：排练好累呀，好想吃巧克力螺！");
    }

    // 2. 抽象方法（留出的空）：大家都会演奏，但怎么演奏？不知道！留给子类自己去实现。
    // 注意这里没有大括号 {}，只有分号 ;
    public abstract void playMusic(); 
}
```
- 例如可以在支付流程中有这样的一个模板
```
// 支付基础模板（抽象类）
public abstract class BasePayment {

    // 这是一个完整的支付流程！
    public void processPayment(double amount) {
        checkBalance(amount); // 1. 查余额（大家做法一样）
        executePay(amount);   // 2. 真正扣钱（大家做法不一样！去填空吧！）
        recordLog();          // 3. 记日志（大家做法一样）
    }

    // 普通方法：查余额的逻辑对于系统内部是一样的
    private void checkBalance(double amount) {
        System.out.println("检查用户账户余额是否大于 " + amount);
    }

    // 普通方法：记录日志写入数据库也是通用的
    private void recordLog() {
        System.out.println("支付完成，将流水写入数据库...");
    }

    // ⭐️ 抽象方法：具体怎么对接微信、支付宝的接口？留给具体的类去写！
    protected abstract void executePay(double amount);
}
```

#### 接口
```
// 只要是消息发送器，就必须能发送消息
public interface MessageSender {
    void sendMessage(String user, String content);
}

// 技能实现者 1：短信发送器
public class SmsSender implements MessageSender {
    @Override
    public void sendMessage(String user, String content) {
        System.out.println("调用阿里云短信接口，发送给 " + user + ": " + content);
    }
}

// 技能实现者 2：邮件发送器
public class EmailSender implements MessageSender {
    @Override
    public void sendMessage(String user, String content) {
        System.out.println("调用腾讯云邮件接口，发送给 " + user + ": " + content);
    }
}
```
#### 两者的区别
- 抽象类强调的是  是什么  的关系，目的在于代码复用；抽象类强调的是 能做什么 的关系，目的在于解决解耦和扩展
- Java 中类只能**单继承**抽象类，但可以**多实现**接口；抽象类是有构造方法的，给子类初始化时调用；而接口没有构造方法；抽象类可以有普通的成员变量，能保存对象的状态；而接口里只能有 `public static final` 的**全局常量**，不能保存状态
### Static 关键字
- static 关键字可以用来修饰变量、方法、代码块和内部类，以及导入包。
- 静态变量：是被Static修饰的变量，属于类，但不属于类的任何一个对象，一个类不管创建多少个对象，静态变量在内存中有且仅有一个副本
- 静态方法：是被Static修饰的方法，它不需要 `new` 出对象，直接用 `类名.方法名` 就可以调用。静态方法里不能访问非静态成员变量和方法
- 静态代码块：只会在类加载的时候才执行一次
### final 关键字
- `final`关键字主要有以下三个方面的作用：用于修饰类、方法和变量。
- 修饰类：当`final`修饰一个类时，表示这个类不能被继承。比如，String 类、Integer 类和其他包装类都是用 final 修饰的。
- 修饰方法：用`final`修饰的方法不能在子类中被重写。
- 修饰变量：当`final`修饰基本数据类型的变量时，该变量一旦被赋值就不能再改变。如果是基本数据类型的变量，其数值一旦在初始化之后就不能更改；如果是引用类型的变量，在对其初始化之后就不能再让其指向另一个对象。

### final finally finalize 的区别
- final是一个修饰符
- finally 是异常处理的一部分，用来创建 try 块后面的 finally 块。无论 try 块中的代码是否抛出异常，finally 块中的代码总是会被执行。
- finalize 是Object 类 的一个方法，用于在垃圾回收器将对象从内存中清除出去之前做一些必要的清理工作。
### == 与 equals 的区别
- = =：用于比较两个对象的引用，即它们是否指向同一个对象实例。如果两个变量引用同一个对象实例，`==` 返回 `true`，否则返回 `false`。
- equals( ) 方法：用于比较两个对象的内容是否相等。默认情况下，`equals()` 方法的行为与 `==` 相同，即比较对象引用，但这个方法时常被各种类重写。
```
String a = new String("沉默王二");
String b = new String("沉默王二");

// 使用 == 比较
System.out.println(a == b); // 输出 false，因为 a 和 b 引用不同的对象

// 使用 equals() 比较
System.out.println(a.equals(b)); // 输出 true，因为 a 和 b 的内容相同
```

### 重写equals 方法的时候必须重写 hashCode？
- hashCode主要是用来获取对象的哈希码，哈希码是由对象的内存地址或者是属性计算出来的，是int类型的整数。哈希码是由哈希函数映射而来的，如果让哈希函数将一个较大的输入域映射到一个较小的输出域，那么就可能出现不同的对象拥有相同的哈希码，也就是发送了哈希冲突。
- 因此，为了解决哈希冲突的问题，哈希表在处理键时，不仅会比较键对象的哈希码，还会使用 equals 方法来检查键对象是否真正相等。如果两个对象的哈希码相同，但通过 equals 方法比较结果为 false，那么这两个对象就不被视为相等。

### 引用类型的变量有什么特点
- 引用类型的变量存储的是对象的地址，而不是对象本身。因此，引用类型的变量在传递时，传递的是对象的地址，也就是说，传递的是引用的值。
