### 怎么理解面向对象？封装继承多态是什么？
- 面向对象就是把现实事物抽象为对象，对象具有属性和行为。
- 封装：把事物的属性和行为结合起来，对外隐藏对象的属性，只保留对象提供的接口供外部交互
- 继承：是一种子类自动共享父类数据结构和方法的机制，是一种代码复用手段
- 多态：是指允许不同类的对象对同意消息做出响应，也就是同一接口，使用不同的实例来执行不同的操作
#### 多态的体现
- 方法重载：同一个类中可以有多个同名方法，各自拥有不同的参数列表，根据传入的参数，编译器会决定使用哪一个方法，例如 `add(int a, int b)` 和 `add(double a, double b)`
- 方法重写：在一个父类中定义一个`Eat` 方法，可以分别在子类`Cat`和`Dog`中分别重写这个方法来实现不一样的操作
- 接口与实现：多个类可以实现同一个接口
- 向上转型和向下转型
	- 向上转型是：**把子类对象赋值给父类类型的引用变量**。
	- ```
	  // 这就是向上转型！把 Guitar 对象，当作 Instrument 来对待。 
	  Instrument myInstrument = new Guitar();
	  ```
	- 也就是 可以使用父类类型的引用指向子类对象
	- 向下转型是： **将父类引用转回其子类类型**
	- ```
	  // 向下转型！把泛指的乐器，强行变回吉他
	  Guitar myGuitar = (Guitar) myInstrument; 
      // 可以用吉他专属的技能
      myGuitar.useWhammyBar();
	  ```
#### 多态的条件
- 子类继承父类
- 子类重写父类的方法
- 父类引用指向子类对象
#### 多态的实现原理是什么
- 多态通过动态绑定实现，Java 使用虚方法表存储方法指针，方法调用时根据对象实际类型从虚方法表查找具体实现。
### 重载和重写的区别
- 如果一个类有多个名字相同但参数个数不同的方法，我们通常称这些方法为方法重载。
- 如果子类具有和父类一样的方法（参数相同、返回类型相同、方法名相同，但方法体不同），我们称之为方法重写。
- 重写是父类和子类中的东西，重载是一个类中的多个同名方法

### 抽象类与接口
#### 抽象类
```
// 这是一个抽象类（半成品模板），不能直接 new 一个 出来哦！
public abstract class BandMember {
    
    String name; // 普通属性：大家都有名字

    public BandMember(String name) {
        this.name = name;
    }

    // 1. 普通方法（已经写好的）：所有成员排练完都会喊累，做法完全一样，直接在模板里写死！
    public void complain() {
        System.out.println(name + " 说：排练好累呀，好想吃巧克力螺！");
    }

    // 2. 抽象方法（留出的空）：大家都会演奏，但怎么演奏？不知道！留给子类自己去实现。
    // 注意这里没有大括号 {}，只有分号 ;
    public abstract void playMusic(); 
}
```
- 例如可以在支付流程中有这样的一个模板
```
// 支付基础模板（抽象类）
public abstract class BasePayment {

    // 这是一个完整的支付流程！
    public void processPayment(double amount) {
        checkBalance(amount); // 1. 查余额（大家做法一样）
        executePay(amount);   // 2. 真正扣钱（大家做法不一样！去填空吧！）
        recordLog();          // 3. 记日志（大家做法一样）
    }

    // 普通方法：查余额的逻辑对于系统内部是一样的
    private void checkBalance(double amount) {
        System.out.println("检查用户账户余额是否大于 " + amount);
    }

    // 普通方法：记录日志写入数据库也是通用的
    private void recordLog() {
        System.out.println("支付完成，将流水写入数据库...");
    }

    // ⭐️ 抽象方法：具体怎么对接微信、支付宝的接口？留给具体的类去写！
    protected abstract void executePay(double amount);
}
```

#### 接口
```
// 只要是消息发送器，就必须能发送消息
public interface MessageSender {
    void sendMessage(String user, String content);
}

// 技能实现者 1：短信发送器
public class SmsSender implements MessageSender {
    @Override
    public void sendMessage(String user, String content) {
        System.out.println("调用阿里云短信接口，发送给 " + user + ": " + content);
    }
}

// 技能实现者 2：邮件发送器
public class EmailSender implements MessageSender {
    @Override
    public void sendMessage(String user, String content) {
        System.out.println("调用腾讯云邮件接口，发送给 " + user + ": " + content);
    }
}
```
